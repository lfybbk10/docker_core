# Запуск приложения

1) Поднять ```postgres-deployment``` и ```postgres-service```
2) Создать ```service-account```
3) Поднять ```vault```
4) Поднять ```configmap``` и ```secret```
5) Поднять само приложение и сервис
6) Можно обращаться к эндпоинтам

Таким образом, мы поднимаем vault в kubernetes, настраиваем на нём аутентификацию для подов с нужным service-account, далее в приложении в конфигурации задаём этот service-account. Теперь приложение может получать все данные из vault, в том числе и динамические креды от поднятой БД. Также приложение имеет локальные configmap и secret для демонстрации обоих подходов.
